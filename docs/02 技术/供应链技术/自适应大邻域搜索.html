2、迭代优化：通过启发式或元启发式算法对当前解进行迭代优化，不断寻找更优的订单分配方案。

3、解的更新：在每次迭代中，如果找到了更优的解，则更新当前解。

4、终止条件：当达到预设的迭代次数、运行时间或解的改进幅度小于阈值时，终止算法。

二、算子 ​

1、算子的设计应该参考改派操作

2、每种类型的算子应该考虑业务倾向（更高的效率）和随机（更广的搜索空间），这样既兼顾搜索效率，也兼顾搜索空间

1、移除算子-随机删除若干订单 ​

随机选取一定数量的订单从路径中删除

2、移除算子-随机删除若干路径 ​

随机选取一定数量的路径从解中删除

3、移除算子-随机删除路径中局部簇订单 ​

随机选择一个 route，然后将route上的顾客分为两个cluster，随机选择一个cluster，然后移除被选择的 cluster 中的订单

4、移除算子-删除目标变坏的若干订单 ​

如果有多个目标，要设计多个目标变化移除算子，删除一定数量对目标值变坏程度比较大的订单

比如最远距离移除：移除当前分配中距离前一单或者前置仓最远的订单（后面所有） 比如最紧时间窗移除：移除剩余时长最短的订单 比如基于骑手类型的移除：根据骑手类型（全职或兼职）移除订单

5、移除算子-超背单上限的移除 ​

在移除订单时，考虑骑手的背单上限。避免移除那些可能导致骑手超过背单上限的订单。

6、移除算子-区域相关性的移除 ​

在移除订单时，考虑订单之间的区域相关性。避免移除那些与其他订单在区域上不相关的订单，以减少突破区域相关性的惩罚。

6、修复算子-增加距离新增最短订单 ​

随机选择一个还没被服务的订单，每次向路径里插入距离增加最小的位置，全局选择订单

6、修复算子-增加贪心距离新增最短订单 ​

按照订单优先级的顺序修复，增加距离最短的骑手

7、修复算子-增加最早送达订单 ​

随机选择一个还没被服务的订单，将订单分配给能够最早送达的骑手，以减少订单剩余时长

8、修复算子-基于骑手效率的分配 ​

根据骑手的效率（如历史配送速度）分配订单，高效率订单分配给全职骑手，低效率订单分配给兼职骑手

8、修复算子-批量修复算子 ​

对于修复算子，如果是逐单修复，很有可能还是分配给之前的骑手，因为修复一个订单，给新骑手更合适；如果修复一批订单，那就不一定了。

所以需要批量修复算子

9、交换算子-区域交换 ​

如果一个破坏和一个修复成对出现，破坏的并不一定能修复上，影响搜索效率；这时候建议增加交换和移交算子

比如将骑手 A 的某个区域订单全部移动骑手 B 上，通常用于有超时风险让一个骑手配送

10、交换算子-指定骑手交换 ​

比如将全职和兼职身上的全部订单交换，通常用于调度分单倾向，全职配送效率高的订单，简直配送效率低的订单

11、交换算子-随机交换 ​

避免总是 A 和 B 交换（优先级目标最好，并不一定合适），可以 A 和 C 交换（优先级目标次优，匹配度次优）

11、平衡骑手工作量 ​

移除那些导致某些骑手工作量过大或过小的订单

12、减少骑手等待 ​

移除那些可能导致骑手工作量不均匀的订单，以探索减少骑手数量的可能性，在重新分配订单时，优先考虑将订单集中分配给一个骑手，直到该骑手的订单达到满载状态。将剩余的订单分配给另一个骑手。

13、避免同小区拆单 ​

优先移除那些会导致同小区订单被拆分的订单，优先考虑将小区内的订单重新分配给同一个骑手

三、目标 ​

用户体验：用户剩余时长 配送效率：骑手往返距离 门店成本：骑手偏好（高效率订单给成本低的骑手）

压力高，提升配送效率权重 压力低，提升用户体验权重

四、停止条件 ​

1、帕累托最优解集的稳定性：连续多次迭代后没有产生新的帕累托最优解

2、收敛性指标：解的分布均匀性

3、多样性：新的迭代没有增加解集的多样性（拥挤度），帕累托前沿中所有解的拥挤距离的平均值

五、修复 ​

1. 随机修复（Random Repair） ​

随机选择一种方式将移除的元素重新插入到解中，通常是以随机的方式选择位置插入。

2. 贪心修复（Greedy Repair） ​

根据某种贪心策略选择插入位置，例如选择插入后成本增加最小的位置。

3. 邻域搜索修复（Local Search Repair） ​

使用邻域搜索方法，在当前解的邻域内搜索，找到一个合适的位置插入元素，以便尽量保持解的质量。

4. 最小插入成本修复（Minimum Insertion Cost Repair） ​

每次选择插入位置，使得插入的额外成本最小化。通常用于TSP和VRP问题中。

5. 分裂修复（Split Repair） ​

将移除的元素进行分组，然后分别插入到解中的不同部分，适用于需要分段处理的问题。

6. 多目标修复（Multi-Objective Repair） ​

考虑多个目标（如成本、时间、距离等），选择插入位置时平衡各个目标，适用于多目标优化问题。

7. 启发式修复（Heuristic Repair） ​

使用特定启发式方法或规则重新插入元素，通常基于问题的具体特征和需求设计。

8. 模拟退火修复（Simulated Annealing Repair） ​

结合模拟退火算法，允许在插入过程中有一定概率接受次优解，以避免陷入局部最优。

9. 遗传算法修复（Genetic Algorithm Repair） ​

结合遗传算法，通过交叉和变异操作来修复解，适用于更复杂的优化问题。

10. 禁忌搜索修复（Tabu Search Repair） ​

结合禁忌搜索算法，记录插入过程中的禁忌操作，避免重复和无效插入，提升解的多样性。

在多目标优化问题中，可以采用不同的修复算子来分别针对每个目标进行优化。这种方法能够更好地平衡多个目标之间的权衡，确保每个目标在修复过程中都得到适当的关注。以下是一些常见的方法：

方法一：权重综合修复 ​

将多个目标通过加权求和的方式综合成一个目标，然后使用综合目标进行修复。

方法二：逐目标修复 ​

针对每个目标分别设计修复算子，依次对每个目标进行修复。

方法三：多目标优化算法 ​

采用专门的多目标优化算法，如NSGA-II、MOEA/D等，在修复过程中同时考虑多个目标。

五、多目标改进算法 ​

自适应大邻域搜索算法（ALNS）主要用于单目标优化问题，但它也可以通过一些扩展和改进来解决多目标问题。下面介绍如何将 ALNS 应用于多目标优化问题：

1. 直接加权法（Weighted Sum Approach） ​

最简单的办法是将多个目标函数通过加权求和的方法合成为一个目标函数。具体步骤如下：

  • 将多个目标函数 f1(x),f2(x),...,fm(x)f_1(x), f_2(x), ..., f_m(x)f1​(x),f2​(x),...,fm​(x) 转换为一个单目标函数：

    F(x)=w1f1(x)+w2f2(x)+...+wmfm(x)F(x) = w_1 f_1(x) + w_2 f_2(x) + ... + w_m f_m(x)F(x)=w1​f1​(x)+w2​f2​(x)+...+wm​fm​(x)

    其中 w1,w2,...,wmw_1, w_2, ..., w_mw1​,w2​,...,wm​ 为不同目标函数的权重。通过调整权重，可以对不同的目标赋予不同的重要性。

  • 然后，使用标准的 ALNS 算法来最小化这个加权和。

优点： ​

  • 简单直接，不需要对 ALNS 算法进行较大改动。

缺点： ​

  • 加权法需要预先确定权重，而不同权重会得到不同的折衷解，难以全面覆盖 Pareto 前沿。

2. 目标分解法（Decomposition-Based Methods） ​

在多目标优化中，目标分解法是一种常见策略。它将多目标问题分解为一系列单目标子问题，然后通过多个优化过程逐步逼近 Pareto 前沿。具体方法包括：

  • 将多目标问题分解成若干个子问题，每个子问题对应一个目标的线性组合（如加权和或切比雪夫函数等）。
  • 在每次搜索中，选择一个子问题并使用 ALNS 进行求解。

这种方法能够有效覆盖 Pareto 前沿中的不同区域。

3. Pareto 优化的扩展（Pareto-based ALNS） ​

另一种更先进的方法是直接在 ALNS 中引入 Pareto 优化的概念，类似于 NSGA-II 中的非支配排序。具体步骤如下：

  • 在每次搜索中，维护一个非支配解集（Pareto 前沿），即解集中没有任何解能够同时在所有目标上支配其他解。
  • ALNS 的邻域搜索过程尝试生成新的解，并将新解加入非支配解集，更新 Pareto 前沿。

具体步骤： ​

  1. 邻域搜索：在每次迭代中，ALNS 使用不同的邻域操作（破坏和修复）生成新解。
  2. Pareto 筛选：对生成的新解，计算其在所有目标函数上的值，并与当前的解集进行比较。根据 Pareto 优势，决定是否将新解加入解集。
  3. 更新解集：如果新解不被其他解支配，则将其添加到解集中；如果新解支配当前解集中的某些解，则替换这些被支配的解。
  4. 继续迭代：不断进行邻域搜索和解集更新，直到满足终止条件。

这种方法允许 ALNS 同时处理多个目标，并找到多个不同的非支配解。

4. 分阶段优化（Sequential Optimization） ​

分阶段优化是一种分步处理多目标的方法：

  • 首先，选择一个主要目标函数进行优化，在该目标函数达到一定阈值后，再优化第二目标，依此类推。
  • 在每个阶段，ALNS 只优化当前目标函数，但随着阶段的进展，逐渐考虑更多的目标函数。

优点： ​

  • 简化了多目标的处理，将问题逐步分解。

缺点： ​

  • 目标的优先级难以调节，且可能会错过一些优秀的 Pareto 折衷解。

5. 目标切换（Objective Switching） ​

目标切换方法在每次迭代中，随机选择一个或多个目标作为当前优化的方向。这种方法类似于多目标模拟退火算法：

  • 在每次迭代时，从多个目标函数中随机选择一个目标（或一组目标）进行优化。
  • 在不同的阶段，算法会偏向于优化不同的目标，最终找到平衡多个目标的解。

优点： ​

  • 目标切换在不修改基本 ALNS 的情况下，可以适应多目标优化的需要。

缺点： ​

  • 优化过程可能不够稳定，因为不同目标的切换可能导致解的震荡。

总结 ​

虽然 ALNS 本身是为单目标优化设计的，但通过适当的扩展，它可以用于解决多目标优化问题。以下是一些常见的扩展方法：

  1. 加权求和法：将多个目标通过加权求和转换为单一目标，适合对权重明确的场景。
  2. 目标分解法：将多目标问题分解为多个单目标子问题。
  3. Pareto 优化的扩展：引入 Pareto 排序和非支配解集，直接在多目标解空间中搜索 Pareto 最优解。
  4. 分阶段优化：逐步处理不同目标，适合需要明确优先级的多目标问题。
  5. 目标切换法：动态切换不同目标进行优化。

要根据具体问题的需求和目标选择合适的扩展方法，如果希望探索广泛的 Pareto 前沿，Pareto 优化的扩展或目标分解法会更加适合。

func (s *AlnsProcessNew) isStoppingCriterionMet() (bool, string) {

	if s.acceptanceCriterion.GetCurrentTemperature() < s.params.MinTemperature {
		return true, "温度过低"
	}

	// 迭代时间
	if time.Now().Unix()-s.beginAt >= int64(s.params.MaxExecDuration) {
		return true, "超时"
	}

	// 收敛性：太多轮没有提升最优解
	if s.status.NIterationWithoutImprovement >= s.params.MaxIterationsNoImprove {
		return true, "解没有提升退出"
	}
	// 收敛性：可行解比较少
	if s.status.NIterationsWithoutNewSolution >= s.params.MaxIterationsNoNew {
		return true, "搜不到新可行解"
	}
	// 收敛性：已搜索全部解
	if len(s.knownKeys) >= s.params.MaxSolutionNum {
		return true, "已搜索全部解"
	}

	// 性能：缓存未命中数限制退出（未命中过多会调用很多路径规划，占用内存过多）
	if s.costCache.GetNoHitTimes() >= s.params.MaxRouteTimes {
		return true, "路径规划次数过多"
	}

	return false, ""
}

func (s *AlnsProcessNew) chooseOperator() (IRepairOperator, IDestroyOperator) {
	repair := s.opManager.SelectRepairOperator()
	var destroy IDestroyOperator
	// 如果是交换算子，则只进行交换（不进行删除）
	if strings.Contains(repair.GetName(), "swap") {
		destroy = nil
	} else {
		destroy = s.opManager.SelectDestroyOperator()
	}

	return repair, destroy
}

func (s *AlnsProcessNew) generateNeighborSolution(repair IRepairOperator, destroy IDestroyOperator) *alg.Solution {
	neighbor := s.currentSolution.CopySolution()
	var removeJobs []alg.Job
	if destroy != nil {
		removeJobs = destroy.DestroySolution(neighbor)
	}
	repair.RepairSolution(neighbor, removeJobs)

	return neighbor
}

	s.status.NIterationRecomputeWeightsSinceLastReload++
	s.status.NIterationWithoutReset++

6、解可行性判断 ​

/**
* 检查当前解是否可行
 */
func (s *AlnsProcessNew) isSolutionAvaliable(newSolution *alg.Solution, riderTypeDispatch bool) bool {

	// 解删除过订单
	if len(newSolution.RemoveJobs) > 0 {
		return false
	}
	// 不满足空间优先级（同区域优先分配相同小哥）
	if newSolution.Objective.SpaceLoss > s.initSolution.Objective.SpaceLoss {
		return false
	}

	// 未开启分层调度，Timeloss 不能大于初始解（弥补时间预估不准的问题）
	if !riderTypeDispatch && newSolution.Objective.TimeLoss > s.initSolution.Objective.TimeLoss {
		return false
	}

	if riderTypeDispatch && newSolution.Objective.TimeLoss > s.initSolution.Objective.TimeLoss+1 {
		return false
	}

	// 超时不能更多
	if newSolution.Objective.UserTimeoutTime > s.initSolution.Objective.UserTimeoutTime {
		return false
	}

	if newSolution.Objective.UserOverTime > s.initSolution.Objective.UserOverTime {
		return false
	}

	// 配送效率优先：不能少于初始解
	// if newSolution.Objective.AssignOrderCount < initObective.AssignOrderCount{
	// 	return false
	// }

	return true
}

func (s *AlnsProcessNew) isImproveCurentSolution(newSolution *alg.Solution) bool {
	if newSolution.SolutionCost == s.currentSolution.SolutionCost {
		return false
	}
	return s.dominates(s.currentSolution, newSolution)
}

/**
* 新解是否支配当前解
* @todo 或者放宽松这里的条件 ε-支配准则，在某些目标上稍微差一些（即差不多在一个容忍度范围内），但在其他目标上有显著优势
 */
func (s *AlnsProcessNew) dominates(solution, newSolution *alg.Solution) bool {
	// B在所有目标上都不比A差，则B支配A
	if solution.Objective.TimePriority >= newSolution.Objective.TimePriority &&
		solution.Objective.RoundTripTime >= newSolution.Objective.RoundTripTime &&
		solution.Objective.DdxValuable <= newSolution.Objective.DdxValuable &&
		solution.Objective.RiderAreaRatio >= newSolution.Objective.RiderAreaRatio {
		return true
	}

	return false
}


/**
* 更新当前解
 */
func (s *AlnsProcessNew) acceptCurentSolution(newSolution *alg.Solution, isImproveCurrent bool) bool {

	// 如果新解支配当前解，如果是则接受新解
	// 如果新解不支配当前解，则有一定概率接受新解
	accepted := s.acceptanceCriterion.TransitionAcceptedMulti(s.currentSolution, newSolution, isImproveCurrent)
	if accepted {
		s.currentSolution = newSolution.CopySolution()
	}

	return accepted
}

/**
* 更新帕累托前沿解
 */
func (s *AlnsProcessNew) updateParetoFront(newSolution *alg.Solution) bool {

	// 将新解与当前的帕累托前沿解集进行比较，并更新前沿
	for key, solution := range s.paretoSolutions {
		// 排除自身解
		if solution.SolutionCost == newSolution.SolutionCost {
			continue
		}

		// 新解支配当前解，从帕累托前沿中移除当前解
		if s.dominates(solution, newSolution) {
			s.dominanceKeys[key] = struct{}{}
			delete(s.paretoSolutions, key)
			continue
		}
		// 如果当前解支配新解，新解不是帕累托最优解
		if s.dominates(newSolution, solution) {
			s.dominanceKeys[newSolution.SolutionCost] = struct{}{}
			delete(s.paretoSolutions, newSolution.SolutionCost)
			return false
		}
	}

	if _, ok := s.paretoSolutions[newSolution.SolutionCost]; ok {
		return false
	} else {
		// 新解不被任何解支配，加入帕累托前沿
		s.paretoSolutions[newSolution.SolutionCost] = newSolution
		return true
	}
}

8、更新状态参数 ​

func (s *AlnsProcessNew) updateParametersByQuality(
	imporveCurrent bool,
	updateCurret bool,
	updateBest bool,
	newSolution *alg.Solution) {

	s.status.NIterationAvaliable += 1

	// 将当前解加入knownkeys
	var ok bool
	if _, ok = s.knownKeys[newSolution.SolutionCost]; !ok {
		s.knownKeys[newSolution.SolutionCost] = struct{}{}
		s.status.AlreadyKnownSolution = FALSE
		s.status.NIterationsWithoutNewSolution = 0
	} else {
		s.status.AlreadyKnownSolution = TRUE
		s.status.NIterationsWithoutNewSolution += 1
	}
	if imporveCurrent {
		s.status.ImproveCurrentSolution = TRUE
		s.status.NIterationWithoutImprovementCurrent = 0
	} else {
		s.status.ImproveCurrentSolution = FALSE
		s.status.NIterationWithoutImprovementCurrent++
	}

	if updateCurret {
		s.status.AcceptedAsCurrentSolution = TRUE
		s.status.NIterationWithoutTransition = 0
		s.status.NIterationUpdateCurrent += 1
	} else {
		s.status.AcceptedAsCurrentSolution = FALSE
		s.status.NIterationWithoutTransition++
	}

	if updateBest {
		s.status.NewBestSolution = TRUE
		s.status.NIterationWithoutImprovement = 0
		s.status.NIterationWithoutImprovementSinceLastReload = 0
		s.status.NIterationUpdateBest += 1
	} else {
		s.status.NewBestSolution = FALSE
		s.status.NIterationWithoutImprovement++
		s.status.NIterationWithoutImprovementSinceLastReload++
	}
}

9、自适应机制 ​

/**
* 自适应策略
 */
func (s *AlnsProcessNew) adapt(repair IRepairOperator, destroy IDestroyOperator) {
	// 更新算子分数
	s.opManager.UpdateScores(destroy, repair, s.status)

	// 更新算子权重（根据算子分数）
	if s.status.NIterationRecomputeWeightsSinceLastReload > s.params.TimeSegmentsIt {
		s.status.NIterationRecomputeWeights += 1
		s.opManager.RecomputeWeights()
		s.status.NIterationRecomputeWeightsSinceLastReload = 0
	}

	// 重置当前解
	if s.status.NIterationWithoutChange > s.params.ResetNbIterationsWithoutChange ||
		s.status.NIterationWithoutReset > s.params.ResetNbIterations {
		s.status.NIterationWithoutChange = 0
		s.status.NIterationWithoutReset = 0
		s.status.NIterationReloadInit += 1
		s.currentSolution = s.initSolution
	}
}

10、非支配排序 ​

// 非支配排序: 根据主目标和次目标的支配关系排名，保留具有较好排名的解
func (s *AlnsProcessNew) sortParetoFront() ([]*alg.Solution, []*alg.Solution) {

	solutions := make([]*alg.Solution, 0, len(s.paretoSolutions)+1)
	paretoSolutions := make([]*alg.Solution, 0, len(s.paretoSolutions))

	// 初始解
	s.paretoSolutions[s.initSolution.SolutionCost] = s.initSolution
	for i, a := range s.paretoSolutions {
		for j, b := range s.paretoSolutions {
			if i != j {
				// 主目标
				if a.Objective.TimePriority > b.Objective.TimePriority {
					a.Objective.Rank += 1
				}
				if a.Objective.RoundTripTime > b.Objective.RoundTripTime {
					a.Objective.Rank += 1
				}
				if a.Objective.DdxValuable < b.Objective.DdxValuable {
					a.Objective.Rank += 1
				}
				// 次目标
				if a.Objective.RiderAreaRatio > b.Objective.RiderAreaRatio {
					a.Objective.Rank += 1
				}
				s.paretoSolutions[i] = a
			}
		}
	}
	for i, solution := range s.paretoSolutions {
		if solution.SolutionCost != s.initSolution.SolutionCost {
			paretoSolutions = append(paretoSolutions, s.paretoSolutions[i])
		}
	}

	// 帕累托解比较多时，时间排在前面，挑选质量好的解
	sort.SliceStable(paretoSolutions, func(i int, j int) bool {
		if paretoSolutions[i].Objective.Rank == paretoSolutions[j].Objective.Rank {
			if paretoSolutions[i].Objective.TimePriority == paretoSolutions[j].Objective.TimePriority {
				return paretoSolutions[i].Objective.RoundTripTime < paretoSolutions[j].Objective.RoundTripTime
			}
			return paretoSolutions[i].Objective.TimePriority < paretoSolutions[j].Objective.TimePriority
		}

		return paretoSolutions[i].Objective.Rank < paretoSolutions[j].Objective.Rank
	})

	// 初始解放到第一个
	solutions = append(solutions, s.initSolution)
	solutions = append(solutions, paretoSolutions...)

	// 召回候选解
	n := len(solutions)
	if n > 8 {
		n = 8
	}
	solutions2 := make([]*alg.Solution, n)
	// 按照非支配排序后，只考虑前 8 个解
	copy(solutions2, solutions[:n])

	return solutions, solutions2
}

11、选最优解 ​

根据场景，找准主目标，主目标最优方案存在不顺路，再进行兜底：主目标上下浮动一定范围内容，顺路程度最高的方案

非分层调度排序：

// 高压（存在超时）：主目标用户体验优先（剩余时间） // 低压：主目标配送效率优先（单均往返时长）

分层调度排序：在不影响履约的前提下进行偏好倾向

// 召回：过滤超时风险相比初始解高的方案 // 高压（存在超时）：主目标匹配度、用户体验优先（剩余时间） // 低压：主目标匹配度、配送效率优先（单均往返时长）

兜底：

如果都顺路的前提下，就是单一目标（用户体验、配送效率、分层调度）选最优；如果不顺路，不允许突破相关性 & 但突破了，就选择N个主目标相比当前最优解在一定范围内，突破相关性最少的方案

func (s *AlnsProcessNew) selectBestSolution(solutions []*alg.Solution, isHighPressure bool, riderTypeDispatch bool) (bestSolution *alg.Solution, bestSolutionType int) {

	// 压力高或者有超时均当作高压场景处理
	minUserOverTime := s.initSolution.Objective.UserOverTime

	// 兜底
	defer func() {
		// 当前未达到平峰期突破相关性策略条件 & 突破相关性
		// 晚上8点后允许突破相关性
		if !s.params.IsTestMode && bestSolution.Objective.RiderAreaRatio > 0 {
			if riderTypeDispatch {
				if isHighPressure {
					bestSolutionType = alg.SOLUTION_TYPE_3
					bestSolution = s.bestSolutionByDdxValuable1(solutions, bestSolution)
				} else {
					bestSolutionType = alg.SOLUTION_TYPE_4
					bestSolution = s.bestSolutionByDdxValuable2(solutions, bestSolution)
				}
			} else {
				if isHighPressure {
					bestSolutionType = alg.SOLUTION_TYPE_5
					bestSolution = s.bestSolutionByTimePriority(solutions, bestSolution)
				} else {
					bestSolutionType = alg.SOLUTION_TYPE_6
					bestSolution = s.bestSolutionByRoundTripTime(solutions, bestSolution)
				}
			}
		}
		// @todo 突破背单上限

		for i, r := range bestSolution.Results {
			bestSolution.Results[i].Rider.Cost = s.costCache.OneResultCostFunc(r)
		}
		bestSolution.SolutionImprove = bestSolution.SolutionCost - s.initSolution.SolutionCost
		bestSolution.Duration = time.Now().Unix() - s.beginAt
		bestSolution.Iterations = s.status.GetIterations()
		bestSolution.IterationsWithoutImprovement = s.status.GetIterationsWithoutImprovement()
	}()

	// 不同场景选择最优策略的方式不一样
	if isHighPressure {
		sort.SliceStable(solutions, func(i, j int) bool {
			if solutions[i].Objective.TimePriority == solutions[j].Objective.TimePriority {
				if solutions[i].Objective.RoundTripTime == solutions[j].Objective.RoundTripTime {
					return solutions[i].Objective.RiderAreaRatio < solutions[j].Objective.RiderAreaRatio
				}
				return solutions[i].Objective.RoundTripTime < solutions[j].Objective.RoundTripTime
			}

			return solutions[i].Objective.TimePriority < solutions[j].Objective.TimePriority
		})
	} else {
		sort.SliceStable(solutions, func(i, j int) bool {
			if solutions[i].Objective.RoundTripTime == solutions[j].Objective.RoundTripTime {
				if solutions[i].Objective.TimePriority == solutions[j].Objective.TimePriority {
					return solutions[i].Objective.RiderAreaRatio < solutions[j].Objective.RiderAreaRatio
				}
				return solutions[i].Objective.TimePriority < solutions[j].Objective.TimePriority
			}

			return solutions[i].Objective.RoundTripTime < solutions[j].Objective.RoundTripTime
		})
	}

	if !riderTypeDispatch || len(solutions) == 1 {
		bestSolutionType = alg.SOLUTION_TYPE_DEFAULT
		bestSolution = solutions[0]
		return
	}

	// 超时风险没有变差的方案中选择匹配度最高的方案
	solutions2 := make([]*alg.Solution, 0, len(solutions))
	for _, solution := range solutions {
		// @todo 预估准确的时候换成超时时长，不准确的时候换成超时风险 badcase52
		if solution.Objective.UserOverTime <= minUserOverTime {
			solutions2 = append(solutions2, solution)
		}
	}

	if isHighPressure {
		sort.SliceStable(solutions2, func(i, j int) bool {
			if solutions2[i].Objective.DdxValuable == solutions2[j].Objective.DdxValuable {
				if solutions2[i].Objective.TimePriority == solutions2[j].Objective.TimePriority {
					return solutions2[i].Objective.RoundTripTime < solutions2[j].Objective.RoundTripTime
				}
				return solutions2[i].Objective.TimePriority < solutions2[j].Objective.TimePriority
			}

			return solutions2[i].Objective.DdxValuable > solutions2[j].Objective.DdxValuable
		})
	} else {
		sort.SliceStable(solutions2, func(i, j int) bool {
			if solutions2[i].Objective.DdxValuable == solutions2[j].Objective.DdxValuable {
				if solutions2[i].Objective.RoundTripTime == solutions2[j].Objective.RoundTripTime {
					return solutions2[i].Objective.TimePriority < solutions2[j].Objective.TimePriority
				}
				return solutions2[i].Objective.RoundTripTime < solutions2[j].Objective.RoundTripTime
			}

			return solutions2[i].Objective.DdxValuable > solutions2[j].Objective.DdxValuable
		})
	}

	bestSolutionType = alg.SOLUTION_TYPE_DDX
	bestSolution = solutions2[0]
	return
}
